## PROJECT OVERVIEW

Build a fully self-hosted, plugin-based Discord moderation bot called **Nightpigeon** written entirely in **Python**. The bot is inspired by Zeppelin (zeppelin.gg) in architecture and configuration philosophy but is a completely original project. It uses YAML-based per-server configuration stored in a PostgreSQL database hosted on Aiven. The bot and its API backend are hosted together as a single Render Web Service. The web dashboard is hosted as a separate Render Static Site. There is NO whitelist system — the bot joins any server freely.

---

## DEPLOYMENT ARCHITECTURE

You will create exactly **2 Render services**:

### Service 1 — Bot + API (Render Web Service)
- Runs the Python Discord bot (`bot.py`) AND a FastAPI HTTP server in the same process using `asyncio`
- The FastAPI server exposes REST endpoints that the dashboard calls for reading/writing YAML configs and viewing cases
- The bot connects to Aiven PostgreSQL using `asyncpg`
- The FastAPI server also uses `asyncpg` (same connection pool)
- Render Web Service type, Python runtime, pinned to Python 3.12 (add `.python-version` file to repo root containing just `3.12`)
- Start command: `python main.py`
- The API listens on `0.0.0.0:$PORT` (Render sets `PORT` automatically)
- Add a `/ping` endpoint that returns `{"status": "ok"}` for UptimeRobot keepalive

### Service 2 — Dashboard (Render Static Site)
- A standalone single-page web app built with plain HTML, CSS, and vanilla JavaScript (no React, no Node build step, no npm required)
- Communicates with Service 1's FastAPI API via `fetch()` calls
- Served as a static site on Render
- Build command: none (static files, just set publish directory to `dashboard/`)
- All dashboard files live in a `dashboard/` folder in the repo

## DATABASE SETUP (Aiven PostgreSQL via asyncpg)

On startup, `database.py` must run all `CREATE TABLE IF NOT EXISTS` statements. The bot creates its own tables — no manual SQL required.

```sql
-- Per-server YAML configuration
CREATE TABLE IF NOT EXISTS guild_configs (
  guild_id BIGINT PRIMARY KEY,
  config TEXT NOT NULL DEFAULT ''
);

-- All moderation cases (warns, bans, kicks, mutes, notes, etc.)
CREATE TABLE IF NOT EXISTS cases (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  case_number INT NOT NULL,         -- per-guild case number, auto-incremented
  user_id BIGINT NOT NULL,
  user_tag TEXT,                     -- cached username#discriminator at time of action
  moderator_id BIGINT NOT NULL,
  moderator_tag TEXT,
  action TEXT NOT NULL,              -- ban, forceban, unban, kick, mute, unmute, warn, note, addcase
  reason TEXT DEFAULT 'No reason provided',
  duration TEXT DEFAULT NULL,        -- e.g. "7d", "1h", "permanent"
  expires_at TIMESTAMP DEFAULT NULL, -- for timed bans/mutes
  created_at TIMESTAMP DEFAULT NOW(),
  active BOOLEAN DEFAULT TRUE        -- false when ban/mute expires
);

-- Tags (custom commands)
CREATE TABLE IF NOT EXISTS tags (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  name TEXT NOT NULL,
  content TEXT NOT NULL,
  creator_id BIGINT,
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(guild_id, name)
);

-- Muted users (for role restore on unmute)
CREATE TABLE IF NOT EXISTS muted_users (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  removed_roles BIGINT[],            -- role IDs that were removed on mute
  expires_at TIMESTAMP DEFAULT NULL,
  UNIQUE(guild_id, user_id)
);

-- Timed bans (for auto-unban)
CREATE TABLE IF NOT EXISTS timed_bans (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  expires_at TIMESTAMP NOT NULL,
  UNIQUE(guild_id, user_id)
);

-- Reminders
CREATE TABLE IF NOT EXISTS reminders (
  id SERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL,
  guild_id BIGINT,
  channel_id BIGINT,
  message TEXT NOT NULL,
  remind_at TIMESTAMP NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Starboard
CREATE TABLE IF NOT EXISTS starboard_entries (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  original_message_id BIGINT NOT NULL UNIQUE,
  starboard_message_id BIGINT,
  star_count INT DEFAULT 0,
  channel_id BIGINT,
  author_id BIGINT
);

-- Reaction roles
CREATE TABLE IF NOT EXISTS reaction_roles (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  message_id BIGINT NOT NULL,
  channel_id BIGINT NOT NULL,
  type TEXT NOT NULL,                -- emoji | button | dropdown
  identifier TEXT NOT NULL,          -- emoji string, button custom_id, or select option value
  role_id BIGINT NOT NULL
);

-- Tickets
CREATE TABLE IF NOT EXISTS tickets (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  channel_id BIGINT NOT NULL UNIQUE,
  user_id BIGINT NOT NULL,
  ticket_number INT NOT NULL,
  status TEXT DEFAULT 'open',        -- open | closed | deleted
  claimed_by BIGINT DEFAULT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Ticket blacklist
CREATE TABLE IF NOT EXISTS ticket_blacklist (
  guild_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  reason TEXT,
  PRIMARY KEY (guild_id, user_id)
);

-- Timezones
CREATE TABLE IF NOT EXISTS user_timezones (
  user_id BIGINT PRIMARY KEY,
  timezone TEXT NOT NULL
);

-- Auto-reply rules
CREATE TABLE IF NOT EXISTS auto_replies (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  trigger TEXT NOT NULL,
  response TEXT NOT NULL,
  trigger_type TEXT DEFAULT 'contains',  -- contains | exact | startswith | endswith | regex
  match_case BOOLEAN DEFAULT FALSE,
  reply_type TEXT DEFAULT 'message',     -- message | reply | dm
  delete_trigger BOOLEAN DEFAULT FALSE,
  delete_after INT DEFAULT NULL,
  ignore_roles BIGINT[] DEFAULT '{}',
  ignore_channels BIGINT[] DEFAULT '{}',
  required_roles BIGINT[] DEFAULT '{}',
  required_channels BIGINT[] DEFAULT '{}'
);

-- Auto-reactions
CREATE TABLE IF NOT EXISTS auto_reactions (
  id SERIAL PRIMARY KEY,
  guild_id BIGINT NOT NULL,
  trigger TEXT NOT NULL,
  trigger_type TEXT DEFAULT 'contains',
  emojis TEXT[] NOT NULL,
  ignore_channels BIGINT[] DEFAULT '{}',
  required_channels BIGINT[] DEFAULT '{}'
);

-- Auto channel clean schedule
CREATE TABLE IF NOT EXISTS auto_channel_clean (
  guild_id BIGINT NOT NULL,
  channel_id BIGINT NOT NULL,
  interval_seconds INT NOT NULL,
  last_cleaned TIMESTAMP DEFAULT NOW(),
  PRIMARY KEY (guild_id, channel_id)
);

-- Dashboard sessions (JWT-based, stored in memory — no DB table needed)
-- OAuth state nonces
CREATE TABLE IF NOT EXISTS oauth_states (
  state TEXT PRIMARY KEY,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## CORE ARCHITECTURE

### `main.py`
Starts both the Discord bot and the FastAPI server in the same asyncio event loop using `asyncio.gather()`. Use `uvicorn` programmatically with `uvicorn.Config` and `uvicorn.Server` so both can share the same loop.

### `bot/core/config_loader.py`
- `async def get_config(guild_id: int) -> dict` — fetches raw YAML text from `guild_configs` table, parses with `yaml.safe_load()`, returns dict. Returns `{}` if no config exists.
- `async def save_config(guild_id: int, config_text: str)` — validates YAML (raises ValueError if invalid), saves to DB.
- Config is NOT cached in memory. Every config read hits the DB. This ensures the dashboard and bot always use the same config.

### `bot/core/level_check.py`
The level system is a permission hierarchy defined in the YAML. Levels are numbers 1–100. Bot owner is always level 1000.

Levels can be assigned to:
- Specific user IDs
- Specific role IDs

The bot takes the **highest level** found across all of a user's role IDs and their direct user ID entry.

```yaml
# In guild YAML config — levels section
levels:
  users:
    "123456789012345678": 100   # This user ID has level 100
    "987654321098765432": 50    # This user ID has level 50
  roles:
    "111111111111111111": 75    # Everyone with this role has level 75
    "222222222222222222": 25    # Everyone with this role has level 25
```

`level_check.py` must expose:
- `async def get_user_level(guild_id, user_id, member) -> int` — checks bot owner first (returns 1000), then checks YAML levels.users and levels.roles and returns the highest match, or 0 if none found.
- `def require_level(level: int)` — a decorator for bot commands that checks the invoking user's level and responds with the configured `no_permission` message if they don't meet the requirement.

### `bot/core/message_formatter.py`
ALL bot responses go through this module. Default format is **plain text**. Embeds are opt-in via YAML.

The formatter resolves a message key from the guild's YAML config. If the key exists and is a string, it's used as plain text with variable substitution. If the key is a dict with an `embed:` key, it builds a `discord.Embed`. If the key doesn't exist in YAML, it falls back to the hardcoded defaults in this file.

Variable substitution uses Python `.format()` with named keys: `{user}`, `{server}`, `{reason}`, `{duration}`, `{case}`, `{mod}`, `{count}`, etc.

**Hardcoded plain text defaults (used when no YAML override exists):**

```python
DEFAULTS = {

  "ban_reply":        "{user} has been banned | Case: {case}",
  "forceban_reply":   "{user} has been forcebanned | Case: {case}",
  "unban_reply":      "{user} has been unbanned | Case: {case}",
  "kick_reply":       "{user} has been kicked | Case: {case}",
  "mute_reply":       "{user} has been muted | Case: {case}",
  "unmute_reply":     "{user} has been unmuted | Case: {case}",
  "warn_reply":       "{user} has been warned | Case: {case}",
  "addcase_reply":    "Case added for {user} | Case: {case}",
  "note_reply":       "Note added for {user} | Case: {case}",
  "deletecase_reply": "Case {case} deleted",
  "editcase_reply":   "Case {case} updated",

  # Moderation — DMs

  "forceban_dm":      "You have been banned from {server} for the following reason: {reason} | Duration: {duration}",
  "kick_dm":          "You have been kicked from {server} for the following reason: {reason}",
```

## DURATION PARSING

All timed actions (ban, mute, temprole) use a shared `parse_duration(text: str) -> timedelta | None` function in `bot/core/duration.py`.

Supported formats: `30s`, `5m`, `2h`, `7d`, `1w`, `1mo` (30 days), `1y` (365 days).

**Maximum duration cap: 180 days.** Any duration over 180 days is rejected with an error message. The word `permanent` or `perm` returns `None` (meaning no expiry).

Every plugin is a `discord.ext.commands.Cog` loaded with `await bot.load_extension(...)` from `main.py`. Every plugin reads its config from `await get_config(ctx.guild.id)` at command invocation time (never cached). Every command goes through `require_level()`. Every response goes through `message_formatter.py`.

### PLUGIN 1: `plugins/moderation.py`

All moderation commands. Every action that affects a user creates a case in the `cases` table and increments the guild's per-guild case counter. Every action sends a DM to the affected user (unless `dm_on_action: false` in their plugin config). Every action posts a channel reply.

**Commands:**

- `!ban @user [duration] [reason]`
  - Bans the user from the server
  - Optional timed ban: e.g. `!ban @user 7d Spamming` — auto-unbans after 7 days
  - Max duration 180 days
  - Creates ban case in DB
  - Inserts into `timed_bans` if duration set
  - Sends ban DM before banning
  - Posts ban_reply in channel

- `!forceban <user_id> [duration] [reason]`
  - Bans a user by ID even if they are not in the server
  - Uses `guild.ban(discord.Object(id=user_id))`
  - Same behavior as ban otherwise

- `!unban <user_id_or_tag> [reason]`
  - Unbans the user
  - Removes from `timed_bans` if present
  - Creates unban case

- `!kick @user [reason]`
  - Kicks the user
  - Sends kick DM before kicking
  - Creates kick case

- `!mute @user [duration] [reason]`
  - Applies the mute role (configured in YAML as `mute_role: ROLE_ID`)
  - Before adding mute role, saves the user's current roles (excluding @everyone and the mute role) to `muted_users.removed_roles`
  - Removes all non-essential roles from the user (configurable: `remove_roles_on_mute: true`)
  - If duration set, stores expiry in `muted_users.expires_at`
  - Sends mute DM
  - Creates mute case

- `!unmute @user [reason]`
  - Removes the mute role
  - Restores all roles that were saved in `muted_users.removed_roles`
  - Deletes the `muted_users` row
  - Sends unmute DM
  - Creates unmute case

- `!warn @user [reason]`
  - Creates a warn case
  - Sends warn DM
  - Posts warn_reply

- `!addcase @user <action> [reason]`
  - Creates a case silently (no DM to user)
  - Supported actions: warn, note, kick, ban, mute
  - Posts addcase_reply

- `!editcase <case_id> reason <new_reason>` or `!editcase <case_id> duration <new_duration>`
  - Updates the reason or duration on an existing case
  - Posts editcase_reply

- `!deletecase <case_id>`
  - Soft-deletes a case (marks it inactive, does not remove from DB)
  - Posts deletecase_reply

- `!reason <case_id> <new_reason>`
  - Shortcut to update just the reason on a case

- `!purge <count> [@user]`
  - Deletes up to 100 messages in the current channel
  - If `@user` given, deletes only that user's messages within the last 100 messages
  - Posts purge_reply

- `!slowmode <#channel> <duration_or_0>`
  - Sets slowmode on the specified channel
  - `!slowmode #general 5s` — 5 second slowmode
  - `!slowmode #general 0` — removes slowmode
  - Posts slowmode_reply

**YAML Config for moderation plugin:**

```yaml
plugins:
  moderation:
    config:
      mute_role: 123456789012345678          # Role ID to apply on mute
      remove_roles_on_mute: true             # Remove all roles when muting (restored on unmute)
      dm_on_action: true                     # DM the user on all actions
      required_level: 50                     # Default level to use any moderation command
      # Per-command level overrides:
      levels:
        ban: 60
        forceban: 70
        unban: 60
        kick: 50
        mute: 50
        unmute: 50
        warn: 25
        addcase: 25
        editcase: 50
        deletecase: 75
        purge: 25
        slowmode: 50
    messages:
      ban_reply: "{user} has been banned | Case: {case}"
      ban_dm: "You have been banned from {server} for the following reason: {reason} | Duration: {duration}"
      # ... (all other message keys override defaults from message_formatter.py)
```

### PLUGIN 2: `plugins/mass_actions.py`

All mass moderation commands. Each action runs in a loop, creates individual cases for each user, and posts a single summary reply at the end.

**Commands:**

- `!masswarn @user1 @user2 ... [reason]` — warns multiple users at once
- `!massmute @user1 @user2 ... [duration] [reason]` — mutes multiple users
- `!masskick @user1 @user2 ... [reason]` — kicks multiple users
- `!massban @user1 @user2 ... [duration] [reason]` — bans multiple users
- `!massunban <id1> <id2> ... [reason]` — unbans multiple user IDs

Each mass command:
- Processes up to 20 users at once (configurable in YAML as `mass_limit`)
- Posts a progress message, then edits it with results
- Creates individual cases for each user
- DMs each user (if dm_on_action enabled)
- Posts one summary: `Mass {action} complete. Success: {n}, Failed: {n}`

### PLUGIN 6: `plugins/automod.py`

Automatic moderation rules. Each rule is toggled and configured independently in YAML. When a rule triggers, it can delete the message, warn the user, mute the user, kick the user, or ban the user — configurable per rule.

**Rules:**

1. **spam** — detects X messages within Y seconds from one user
2. **bad_words** — detects words from a configurable blacklist (supports wildcards like `*fuck*`)
3. **invite_links** — detects Discord invite links (`discord.gg/...`)
4. **mass_mentions** — detects messages mentioning more than X users/roles
5. **caps_spam** — detects messages that are more than X% uppercase (min length threshold)
6. **link_spam** — detects messages with more than X URLs
7. **repeated_text** — detects the same message sent X times in Y seconds by the same user
8. **mass_emojis** — detects messages with more than X emojis
9. **zalgo** — detects zalgo/corrupted text using Unicode combining character detection
10. **newline_spam** — detects messages with more than X newlines
11. **new_account_gate** — flags or actions users whose Discord account is younger than X days
12. **attachment_spam** — detects sending X+ attachments in Y seconds
13. **mention_everyone_attempt** — detects attempts to @everyone even without permission

**Actions per rule (configurable):**
- `delete: true` — delete the triggering message
- `warn: true` — issue a warn case
- `mute: "10m"` — mute for duration
- `kick: true` — kick from server
- `ban: "7d"` — ban for duration
- `log_channel: CHANNEL_ID` — log the trigger to a channel
- `dm_user: true` — DM the user with the configured automod message
- `reply: true` — post a message in channel

**YAML Config:**
```yaml
plugins:
```

## API SERVER (`api/server.py`)

FastAPI app running inside the same process as the bot. Use `asyncio` + `uvicorn` to run both together in `main.py`.

**CORS:** Only allow origins listed in `ALLOWED_ORIGINS` env var.

**Auth:** Discord OAuth2. Session stored as a JWT cookie.

**Endpoints:**

```
GET  /ping                          → {"status": "ok"}  (for UptimeRobot)

GET  /api/auth/login                → Redirect to Discord OAuth2 authorization URL
GET  /api/auth/callback             → Handle OAuth2 callback, issue JWT session cookie
GET  /api/auth/logout               → Clear session cookie
GET  /api/auth/me                   → Return current logged-in Discord user info

GET  /api/guilds                    → List guilds the logged-in user shares with the bot
GET  /api/guilds/{guild_id}/config  → Get raw YAML config text for a guild
PUT  /api/guilds/{guild_id}/config  → Save new YAML config text (validates YAML first)
GET  /api/guilds/{guild_id}/cases   → Get cases for a guild (paginated, filterable)
GET  /api/guilds/{guild_id}/cases/{case_id} → Get one case by ID
```

The `PUT /api/guilds/{guild_id}/config` endpoint:
1. Verifies the logged-in user is in the guild AND their level in the guild is ≥ 100 (admin-tier)
2. Parses the YAML to validate it (returns 400 if invalid)
3. Saves to `guild_configs` table
4. Returns `{"success": true}`

## FINAL NOTES FOR REPLIT AGENT

1. Every command in every plugin MUST respect the level system. Use `require_level()` from `level_check.py` on every command.
2. Every message the bot sends MUST go through `message_formatter.py`. Never `ctx.send()` a hardcoded string directly from a plugin file.
3. The config is NEVER cached. Every command reads from the DB fresh. This is intentional so dashboard edits take effect immediately.
4. Build the entire project in one go. Do not skip any plugin or feature.
5. The bot prefix is configurable in YAML as `prefix: "*"` at the root of each guild's config. Default to `!` if not set.
6. All paginated embeds/messages use ◀ ▶ emoji reactions for navigation.
7. Duration cap is 180 days. Any duration over 180 days must be rejected with an error message.
8. Mute restores roles on unmute. Store the removed roles in `muted_users.removed_roles` (BIGINT array).
9. The bot handles `on_guild_join` — but does NOT leave. It simply sets up the database row for the guild if it doesn't exist yet.
10. All DB operations use `asyncpg`. Never use `psycopg2` in the bot or API.
11. The dashboard uses plain vanilla JavaScript only — no React, no Vue, no build step.
12. The pigeon/night sky theme must be applied to EVERY page in the dashboard directory.
